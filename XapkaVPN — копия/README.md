ХапкаVPN Telegram Бот
Бот создан для проекта ХапкаVPN — простого VPN-сервиса от студентов из Химок.Позволяет пользователю активировать подписку, продлить ее, проверить статус и получить VLESS ключ.Разработан на Python с использованием библиотеки aiogram 3.x.

Функциональность

Приветствие и главное меню: Пользователь получает приветственное сообщение и меню с опциями: "Активировать подписку", "Продлить подписку", "Статус подписки", "VLESS ключ".
Выбор тарифа: Пользователь может выбрать подписку на 1, 2 или 3 месяца с подтверждением выбора и возможностью вернуться к тарифам или в главное меню.
Имитация оплаты: Заглушка для обработки платежей с отображением корректного склонения ("1 месяц", "2 месяца", "3 месяца").
Продление подписки: Проверка статуса подписки с разветвлением:
Если подписка активна: предложение продлить с подтверждением.
Если подписка неактивна: предложение оформить новую подписку.


Статус подписки: Отображение имени пользователя, статуса (активирована/не активирована) и даты окончания подписки с возможностью перейти к VLESS ключу.
VLESS ключ: Отображение ключа в чистом виде для удобного копирования (заглушка vless://username).
Навигация: Кнопки "В главное меню" и "Вернуться к тарифам" для удобного перемещения по интерфейсу.
Будущие доработки: Подключение к SQL для хранения данных о подписках и интеграция с реальной системой генерации VLESS ключей.


Структура проекта
XapkaVPN/
├── bot/
│   ├── __init__.py        # Пустой файл для обозначения пакета
│   ├── config.py          # Загрузка токена бота из .env
│   ├── handlers.py        # Логика обработки команд и callback-запросов
│   ├── keyboards.py       # Определение клавиатур Telegram
│   ├── messages.py        # Текстовые шаблоны сообщений
│   └── services.py        # Логика получения статуса подписки (заглушка для SQL)
├── main.py                # Точка входа для запуска бота
├── requirements.txt       # Зависимости проекта
├── .env                   # API-токен для бота
└── README.md              # Документация проекта



Текущая реализация использует словарь user_subscriptions для хранения данных о подписках. В будущем планируется переход на SQL.
VLESS ключ формируется как заглушка (vless://username). Реальная генерация ключей будет добавлена позже.
Логика оплаты является заглушкой и будет заменена на реальную интеграцию с платежной системой.
